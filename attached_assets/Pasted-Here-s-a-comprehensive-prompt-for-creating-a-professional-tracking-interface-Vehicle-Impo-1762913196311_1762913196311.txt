# **Vehicle Import Partnership Dashboard - Interface Design Prompt**

## **Project Overview**
Design a clean, professional web-based dashboard to track a vehicle import partnership between Dominick Scipione (Investor) and Tony Hamilton Webster (Dealer Partner). The business imports vehicles from USA to Honduras for resale, with a 60/40 profit distribution during reinvestment phase (until $150K inventory), then 50/50 split.

---

## **Core Features & Modules**

### **1. DASHBOARD HOME (Main Overview)**

**Key Metrics Cards (Top Row):**
- **Total Investment to Date**: Running total of all capital invested
- **Current Inventory Value**: Total cost of unsold vehicles
- **Total Gross Profit**: All-time profit from sold vehicles
- **Progress to $150K Goal**: Progress bar showing path from current inventory to $150K threshold
- **Reinvestment Phase Status**: Badge showing "REINVESTMENT PHASE (60/40)" or "POST-REINVESTMENT (50/50)"

**Quick Stats (Second Row):**
- **Vehicles in Transit**: Count and total value
- **Vehicles in Stock**: Count and total value (in Honduras, unsold)
- **Vehicles Sold**: Count and total revenue
- **Pending Payments**: Amount owed by Tony

**Visual Elements:**
- Line chart showing inventory value growth over time
- Pie chart showing portfolio composition (In Transit / In Stock / Sold)
- Bar chart comparing target prices vs actual sale prices

## **Design Specifications**

### **Color Scheme:**
- **Primary**: Deep Blue (#1E40AF) - Trust, professionalism
- **Secondary**: Emerald Green (#10B981) - Profit, success
- **Accent**: Amber (#F59E0B) - Alerts, pending items
- **Danger**: Red (#EF4444) - Overdue, penalties
- **Neutral**: Gray scale for backgrounds and borders

### **Typography:**
- **Headers**: Inter or SF Pro Display (Bold, 24-32px)
- **Body**: Inter or SF Pro Text (Regular, 14-16px)
- **Numbers/Data**: SF Mono or Roboto Mono (Medium, 14-16px)

### **Layout:**
- **Sidebar Navigation** (left side):
  - Dashboard
  - Shipments
  - Inventory
  - Contracts
  - Financials
  - Payments
  - Costs
  - Reports
  - Settings

- **Top Bar**:
  - Logo / Company name
  - Search bar (global search across all modules)
  - Notifications bell icon
  - User profile dropdown

- **Content Area**: 
  - Clean white background
  - Generous padding and whitespace
  - Card-based layouts for information grouping
  - Responsive tables with sorting/filtering

### **Key UI Components:**

**Status Badges:**
- In Transit: Blue badge with truck icon
- In Stock: Yellow badge with warehouse icon
- Sold: Green badge with checkmark
- Pending Payment: Orange badge with clock icon
- Paid: Green badge with dollar sign
- Overdue: Red badge with warning icon

**Progress Indicators:**
- Linear progress bar for $150K goal
- Circular progress for individual shipment status
- Timeline view for vehicle journey (Purchase â†’ Sale)

**Data Entry Forms:**
- Clean, labeled input fields
- Inline validation
- Auto-calculation of totals where applicable
- File upload with drag-and-drop
- Date pickers for all date fields

**Action Buttons:**
- Primary actions: Solid blue buttons (e.g., "Save", "Submit")
- Secondary actions: Outlined buttons (e.g., "Cancel", "View")
- Danger actions: Red buttons (e.g., "Delete")

## **Special Features**

### **Automated Calculations:**
- Auto-calculate vehicle total cost from all cost components
- Auto-calculate gross profit (Sale Price - Total Cost)
- Auto-calculate profit distribution based on phase (60/40 or 50/50)
- Auto-calculate days in inventory
- Auto-calculate payment due dates (5 business days from sale)

### **Alerts & Notifications:**
- Payment overdue alerts
- Vehicles in inventory >60 days
- Approaching $150K inventory threshold
- Document expiration reminders (insurance, etc.)
- New sale recorded notification

### **Mobile Responsive:**
- Full mobile version for checking status on the go
- Quick access to key metrics
- Mobile-optimized tables (stack on small screens)
- Touch-friendly buttons and controls

### **Data Security:**
- User authentication (login required)
- Role-based access (Dominick sees all, Tony sees relevant sales data)
- Encrypted document storage
- Audit trail (who edited what and when)

## **Example User Flows**

### **Flow 1: Recording a New Shipment**
1. User clicks "Add New Shipment"
2. Enters shipment details (date, route)
3. Adds vehicles to shipment (either one-by-one or bulk import)
4. Uploads cost receipts for each category
5. System auto-calculates total shipment cost
6. User saves shipment
7. Dashboard updates with new inventory value

### **Flow 2: Recording a Vehicle Sale**
1. User navigates to vehicle in inventory
2. Clicks "Mark as Sold"
3. Enters sale information (price, buyer details, date)
4. System calculates profit distribution
5. Generates Phase 3 Sale Closure document
6. Creates pending payment record
7. Sends notification to both partners
8. Updates dashboard metrics

### **Flow 3: Tracking Payment**
1. Tony makes wire transfer
2. Dominick receives payment
3. Navigates to Payments module
4. Finds pending payment record
5. Uploads wire transfer receipt
6. Marks payment as "Paid" with date and reference
7. System updates financial totals
8. Payment moves from Pending to Paid section
